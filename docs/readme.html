What is Agent Antonova?

Agent Antonova is a fully autonomous software development agent that operates continuously without human intervention. It's a Node.js application that integrates with the Claude Code SDK to create a self-managing development workflow.

Core Concept

The agent runs in an infinite loop, automatically discovering work, implementing solutions, and submitting pull requests. It represents a paradigm shift from human-driven to AI-driven development workflows.

Key Characteristics

  • Fully Autonomous: Runs without any human prompts or approvals
  • Continuous Operation: Never stops unless explicitly terminated
  • GitHub Native: Works directly with GitHub issues and pull requests
  • Priority Driven: Uses sophisticated scoring to select work
  • Self-Documenting: Creates comprehensive documentation for all changes

Core Components

1. AutonomousAgent Class

The heart of the system, located in src/autonomous-agent.js:

JavaScript 
export class AutonomousAgent {
  constructor() {
    this.issueEngine = new IssueEngine();
    this.isRunning = false;
    this.currentIssue = null;
    this.executionHistory = [];
    this.loopInterval = 30000; // 30 seconds
  }

  async start() {
    this.isRunning = true;
    while (this.isRunning) {
      try {
        await this.processNextIssue();
      } catch (error) {
        console.error('❌ Error:', error.message);
        await this.sleep(this.loopInterval);
      }
    }
  }
}

Claude Code SDK Integration

What is Claude Code SDK?

The Claude Code SDK (@anthropic-ai/claude-code) is Anthropic's official JavaScript/TypeScript library for programmatic interaction with Claude Code. It enables developers to integrate Claude's coding capabilities directly into their applications.

SDK Context and Purpose

Claude Code SDK provides:

  • Programmatic Access: Use Claude Code without the CLI interface
  • Streaming Responses: Real-time message streaming for long operations
  • Permission Modes: Control over Claude's autonomy level
  • Tool Integration: Access to file operations, shell commands, and more

How Agent Antonova Uses the SDK

JavaScript 
import { query } from "@anthropic-ai/claude-code";

async executeWithClaude(issueContent) {
  const messages = [];
  
  // Create a query with full autonomy
  const stream = query({
    messages: [{
      role: 'user',
      content: this.buildPrompt(issueContent)
    }],
    permissionMode: 'acceptAll',  // Full autonomy
    maxTurns: 15                  // Allow complex operations
  });

  // Process streaming responses
  for await (const chunk of stream) {
    if (chunk.message) {
      messages.push(chunk.message);
      process.stdout.write('.');  // Progress indicator
    }
  }

  return messages;
}

Permission Modes Explained

Mode Description Use Case
acceptAll Claude executes all operations without asking Full automation (Agent Antonova)
acceptEdits Auto-approve edits, ask for other operations Semi-autonomous workflows
askAll Ask permission for every operation Interactive development
plan Only create a plan, don't execute Planning phase workflows

Working Mechanics Deep Dive

The Infinite Loop

Agent Antonova's core is an infinite loop that never stops unless explicitly terminated:

Execution Flow 
while (this.isRunning) {
  // 1. Query for highest priority issue
  const issue = await this.issueEngine.getNextReadyIssue();
  
  if (!issue) {
    await this.sleep(30000);  // Wait 30 seconds
    continue;                 // Try again
  }
  
  // 2. Update issue status
  await this.issueEngine.updateIssueStatus(issue.id, 'in-progress');
  
  // 3. Parse issue structure
  const { goal, acceptanceCriteria, technicalSpecs } = 
    this.parseIssue(issue.body);
  
  // 4. Execute with Claude
  const result = await this.executeWithClaude(issue);
  
  // 5. Create pull request
  await this.createPullRequest(issue, result);
  
  // 6. Update to review status
  await this.issueEngine.updateIssueStatus(issue.id, 'review');
  
  // 7. Brief pause, then continue
  await this.sleep(5000);
}

Issue Processing Pipeline

Step 1: Issue Discovery

The agent queries GitHub for issues matching specific criteria:

  • Must have "ready" status (label or project column)
  • Not assigned to anyone
  • Not in "blocked" status
  • Sorted by priority score (highest first)

Step 2: Issue Parsing

Issues must follow a specific format for optimal processing:

Markdown 
## Goal
Implement a new caching layer for API responses

## Acceptance Criteria
- [ ] Cache implementation using Redis
- [ ] 5-minute TTL for all cached responses
- [ ] Cache invalidation on data updates
- [ ] Performance improvement of at least 50%

## Technical Specs
- Use Redis with node-redis client
- Implement as Express middleware
- Add cache headers to responses
- Include cache hit/miss metrics

Step 3: Autonomous Execution

Claude Code SDK takes over with full autonomy:

What happens during execution:
  • Claude reads the entire codebase context
  • Analyzes existing patterns and conventions
  • Implements the solution following best practices
  • Runs tests and validates the implementation
  • Creates comprehensive commit messages

Step 4: Pull Request Creation

The agent automatically:

  1. Creates a feature branch from main
  2. Commits all changes with detailed messages
  3. Pushes to GitHub
  4. Creates a pull request with:
    • Link to the original issue
    • Summary of changes
    • Test results
    • Any relevant notes

Error Handling and Recovery

The agent is designed to be resilient:

Graceful Degradation

Errors in one issue don't stop the agent - it logs and continues

State Persistence

Execution history is maintained for debugging and analysis

Signal Handling

SIGINT/SIGTERM complete current work before stopping

GitHub Token Requirements

The agent requires specific GitHub token permissions depending on the adapter:

Adapter Required Scopes Purpose
GitHub Labels (default) repo, read:org Full repository access for issues and labels
GitHub Projects project, read:project, repo Projects v2 API access and repository access
Common Permission Issues:
  • "Resource not accessible by personal access token" - Token lacks required permissions
  • Solution: Run gh auth refresh -s repo,read:org for labels adapter
  • Alternative: Create new token at GitHub Settings
Authentication Commands 
# Initial setup
gh auth login

# For GitHub Labels adapter (default)

# For GitHub Projects adapter

Priority Calculation Deep Dive

The priority system uses a sophisticated formula:

Priority Score = (Impact × Urgency × 10) ÷ Effort + Manual Adjustment

Example Calculations:

Critical Bug (High urgency, Small effort):
(40 × 3 × 10) ÷ 2 + 0 = 600 points

New Feature (Medium impact, Normal urgency, Large effort):
(20 × 2 × 10) ÷ 5 + 0 = 80 points

Tech Debt (Low impact, Eventually, Medium effort):
(10 × 1 × 10) ÷ 3 + 0 = 33.33 points
